Troubleshooting y Cheatsheet CLI

Si algo falla en Packet Tracer, no borres todo. Utiliza estos comandos en el modo privilegiado (#) para diagnosticar dónde está el error.

Comando Completo Desglose de Parámetros (Flags) y Función Principal
show ip interface brief Muestra un resumen vital del estado de todos los puertos.
• show: Ordena mostrar datos en pantalla.
• ip: Filtra la información para mostrar solo configuraciones de Capa 3 (IPv4).
• interface: Apunta al hardware (los puertos).
• brief: Parámetro clave. Resume el output en una tabla limpia (IP asignada, estado físico UP/DOWN, y estado de protocolo).
show vlan brief Verifica si creaste y asignaste bien las VLANs en Capa 2.
• vlan: Apunta al archivo interno de la base de datos de VLANs (vlan.dat).
• brief: Filtra la salida para mostrar solo el ID de la VLAN, su nombre y qué puertos exactos de acceso pertenecen a ella.
show interfaces trunk Audita las conexiones entre switches (Enlaces Troncales).
• interfaces: Apunta a los puertos del equipo.
• trunk: Actúa como filtro absoluto. Solo te mostrará los puertos que lograron negociar con éxito el modo troncal y te indicará si todas las VLANs están permitidas para cruzar por ahí.
show ip route Muestra el "GPS" del Router o Switch Multicapa.
• route: Extrae la tabla de enrutamiento interna. Es vital para comprobar si RIPv2 aprendió las rutas de las otras sedes (marcadas con una "R") o si tus rutas estáticas (marcadas con "S") están activas.
¿Para qué sirve el flag "do"?

Normalmente, si estás configurando una IP dentro de una interfaz (config-if)# y quieres hacer un ping o usar un comando show, el sistema te dará error porque esos comandos pertenecen al modo privilegiado #. El parámetro do fuerza al sistema operativo a ejecutar comandos de visualización o guardado sin tener que salirte del modo de configuración.

Comando Aplicación y Efecto
show running-config
(Atajo: sh run)		 Muestra línea por línea toda la configuración que el equipo tiene actualmente en la memoria RAM. Si se apaga el equipo, esta configuración se pierde. Ideal para ver si te equivocaste en una IP o en una política de ACL.
copy running-config startup-config El comando oficial para guardar. Copia todo lo que está en la RAM (running) y lo inyecta en la NVRAM (startup). Así, si Packet Tracer crashea o apagas el rack, tu configuración sobrevive.
do write
(Atajo popular: do wr)		 Hace exactamente lo mismo que el comando anterior (guarda en la NVRAM), pero es mucho más rápido de teclear. Al incluir el flag do, puedes guardar tu progreso en cualquier momento, incluso si estás a mitad de configurar una VLAN o una política de seguridad.
01

Diseño Lógico IP y VLSM

Fundamento Teórico

Las redes "Classful" usan máscaras fijas, desperdiciando miles de IPs. VLSM (Variable Length Subnet Mask) permite "hacer subredes de una subred", asignando máscaras variables según la necesidad exacta de cada departamento, maximizando la eficiencia.

Según la rúbrica, no puedes dimensionar la red con los hosts actuales. Debes aplicar un aumento proporcional del 25% a cada VLAN.

Ejemplo Matemático Exacto (Sede Lima):

Administración: Tienes 100 hosts actuales. Le calculas el aumento del 25% (100 * 1.25) y tienes que dimensionar para125 hosts. Necesitarás una subred /25 (126 direcciones útiles)

Logística: Tienes 25 hosts actuales. Con el aumento serían 31.25. Como no puede haber decimales de host, siempre se redondea hacia arriba, por lo que te quedarías con 32 hosts. Necesitarás una subred /26 (62 útiles).

* Aplica este mismo cálculo y redondeo hacia arriba para absolutamente todos los demás departamentos de la topología.

No desperdicies IPs en Enlaces Seriales

Para los enlaces WAN (Conexiones Punto a Punto entre los Routers Cisco 2911 de cada sede), debes usar SIEMPRE subredes /30. Una máscara /30 solo provee 2 direcciones útiles, exactamente las que necesitas para conectar las interfaces seriales de dos routers. Si usas /24 o FLSM, perderás puntos de eficiencia.

El Núcleo - Enrutamiento Inter-VLAN (Cisco 3650)

El proyecto utiliza el Switch L3 Cisco 3650. Este dispositivo de Capa 3 asume el rol de Gateway para todas las VLANs de las PCs, creando las "Interfaces Virtuales (SVI)".

¡COMANDO CRÍTICO Y OBLIGATORIO!

Por defecto, un Switch Multicapa viene "dormido" de fábrica y solo actúa como un Switch común (Capa 2). Esto significa que aunque crees las VLANs y les pongas IPs, el equipo no sabe cruzar datos de una VLAN a otra.

💡 Analogía simple: Imagina que el switch es un edificio. Crear las VLANs es como construir los pisos, y las SVIs son las puertas de los departamentos. Si no pones el comando ip routing, es como dejar todas las puertas con llave: nadie puede salir de su piso ni visitar a sus vecinos (ningún ping pasará entre VLANs).

¿Qué significa "Modo Global"?
Cisco organiza sus comandos por niveles. El modo global reconoce el prompt CORE(config)#. Los comandos que escribes aquí afectan a todo el equipo por completo. No tienes que poner ip routing antes de cada línea de código; basta con ponerlo una sola vez en este modo para "encender el motor de enrutamiento" de todo el switch antes de probar tus pings.

! PASO 1: ENCIENDE EL MOTOR DEL SWITCH (Modo Global) ! ip: Indica que el comando trabajará con el protocolo IPv4. ! routing: Activa la tabla de enrutamiento interna en todo el hardware del switch L3. CORE(config)# ip routing
! PASO 2: CREAR LOS CONTENEDORES (VLANs) ! Para asegurar compatibilidad total en el Cisco 3650, creamos las VLANs de forma individual en el modo de configuración: CORE(config)# vlan 10 CORE(config-vlan)# vlan 20 CORE(config-vlan)# vlan 30 CORE(config-vlan)# vlan 40 CORE(config-vlan)# vlan 50 CORE(config-vlan)# vlan 60 CORE(config-vlan)# vlan 70 CORE(config-vlan)# vlan 80 CORE(config-vlan)# vlan 90 CORE(config-vlan)# exit
! PASO 3: CONFIGURAR LAS PUERTAS DE ENLACE (SVIs) ! interface vlan: Crea o selecciona la interfaz virtual (SVI) de la VLAN 10 (El prompt cambia al sub-modo específico config-if). CORE(config)# interface vlan 10 ! description: Añade una etiqueta de texto para identificar la red (no afecta el rendimiento). CORE(config-if)# description AdministracionVLAN10 ! ip address [IP] [Máscara]: Asigna la IP fija que usarán los hosts como Gateway. (Nota: Observa la máscara /25 ajustada). CORE(config-if)# ip address 10.81.41.1 255.255.255.128 ! no shutdown: "no" cancela o revierte un comando; "shutdown" mantiene la interfaz apagada. Al combinarlos, se enciende la interfaz. CORE(config-if)# no shutdown
! Repetimos el mismo proceso para las demás VLANs entrando a sus respectivas interfaces virtuales: CORE(config)# interface vlan 20 CORE(config-if)# description LogisticaVLAN20 ! ip address de ejemplo usando subred /26: CORE(config-if)# ip address 10.81.42.129 255.255.255.192 CORE(config-if)# no shutdown
! Nota: Continúa aplicando esta misma lógica para las VLANs restantes usando tus IPs de VLSM ajustadas con el incremento del 25%.

Enlaces WAN (Routers 2911) y Autenticación PPP CHAP

Fundamento Teórico

En enlaces seriales WAN (entre tus routers Cisco 2911), el protocolo por defecto de Cisco es HDLC. Sin embargo, para mayor compatibilidad y seguridad cambiamos el encapsulamiento a PPP (Point-to-Point Protocol).

💡 Analogía simple: Piensa en la autenticación PAP como si enseñaras tu DNI en la puerta del edificio (cualquiera que mire puede ver tus datos, porque va en texto plano). En cambio, CHAP es como usar un token de banco en tu celular; envía un desafío matemático cifrado en 3 vías que cambia constantemente, haciendo casi imposible que un hacker intercepte tu clave. ¡Por eso la rúbrica exige CHAP!
Regla de Nombres de Usuario

Si configuras el ROUTER A que conecta con el ROUTER B, el comando debe ser `username ROUTER_B secret miclave`. La clave (`secret`) debe ser EXACTAMENTE la misma en ambos routers 2911.

! CONFIGURACIÓN EN ROUTER 2911 LIMA (Conectando hacia Ica): Router_Lima(config)# username Router_Ica secret ciscoclass Router_Lima(config)# interface s0/0/0 Router_Lima(config-if)# ip address 10.81.36.1 255.255.255.252 Router_Lima(config-if)# encapsulation ppp Router_Lima(config-if)# ppp authentication chap Router_Lima(config-if)# no shutdown
! CONFIGURACIÓN EN ROUTER 2911 ICA (Conectando hacia Lima): Router_Ica(config)# username Router_Lima secret ciscoclass Router_Ica(config)# interface s0/0/0 Router_Ica(config-if)# ip address 10.81.36.2 255.255.255.252 Router_Ica(config-if)# encapsulation ppp Router_Ica(config-if)# ppp authentication chap Router_Ica(config-if)# no shutdown

Enrutamiento RIPv2 y Rutas Estáticas

Fundamento Teórico

RIPv2 es un protocolo de vector distancia que compartirá automáticamente las rutas entre nuestras sedes.

💡 Analogía simple: Si las VLANs son las oficinas de un edificio y el Switch 3650 es el gerente interno, RIPv2 configurado en los Routers 2911 es el "GPS Corporativo" entre edificios. Le dice a cada sucursal de la empresa cuál es el camino con menos "saltos" para enviar un paquete desde Lima hasta Arequipa sin perderse.

Por otro lado, las Rutas Estáticas Flotantes nos permiten tener un enlace ISP de respaldo hacia Internet. Ese respaldo solo se activará si el enlace principal "se cae" (lo cual logramos configurando una distancia administrativa mayor a 1).

! 1. CONFIGURAR RIPv2 EN TODOS LOS ROUTERS 2911 Y SWITCHES 3650: Router(config)# router rip Router(config-router)# version 2 ! Debes declarar las redes directamente conectadas. Por ejemplo, si el router tiene la 10.81.36.0: Router(config-router)# network 10.81.36.0 Router(config-router)# network 10.81.0.0 ! CRÍTICO: Prevenir que RIP destruya el VLSM sumarizando las redes Router(config-router)# no auto-summary Router(config-router)# exit
! 2. RUTAS ESTÁTICAS AL INTERNET (Solo en el Router 2911 Frontera / Borde): ! Ruta Predeterminada Principal por el ISP 1 Router(config)# ip route 0.0.0.0 0.0.0.0 209.165.200.226 ! Ruta Flotante de Respaldo por el ISP 2 (Distancia Administrativa 5) Router(config)# ip route 0.0.0.0 0.0.0.0 209.165.201.2 5

Servicios de Red (La Magia de Capa 7)

DNS (Domain Name System)

Las computadoras se comunican usando IPs numéricas, pero los humanos usamos nombres (ej. `www.google.com`). El servidor DNS actúa como una gigantesca guía telefónica que traduce estos nombres a IPs.

WEB (HTTP/HTTPS)

El Servidor WEB procesa aplicaciones del lado del servidor. Generalmente usa el protocolo HTTP (puerto TCP 80) o HTTPS (puerto TCP 443) para enviar el código de la página al navegador del cliente.

CORREO (SMTP, POP3, IMAP)

Trabaja como una oficina postal. El protocolo SMTP (Puerto TCP 25) se encarga de transportar o enviar el mensaje (MTA). El protocolo POP3 (Puerto TCP 110) se encarga de descargar y recuperar el correo desde el servidor hacia la PC cliente. IMAP es similar pero mantiene la copia sincronizada en el servidor.

FTP y TCP

FTP permite la transferencia de archivos. Utiliza el protocolo TCP, el cual orienta la conexión y garantiza la fiabilidad del envío de los paquetes (a diferencia de UDP que es más rápido pero no garantiza la entrega).

Seguridad (Las 5 Políticas ACL y SSH)

No copies a ciegas: Entiende los parámetros

En ciberseguridad, un parámetro o "flag" mal colocado puede dejar expuesta la red entera. Por lo tanto, este es el significado exacto de los que usaremos en el código:

• any y host: any engloba a "cualquier origen o destino". Por otro lado, host restringe la regla a una IP única y exacta (ideal para servidores).

• eq (Equal): Le indica a la ACL que intercepte un paquete cuyo protocolo coincida "exactamente" con el número de puerto especificado (ej. eq 80 para la web).

• modulus: Al generar la llave criptográfica (crypto key generate rsa), define el tamaño matemático de tu cifrado SSH.

• transport input ssh: Mata de raíz las conexiones vulnerables. Rechaza protocolos de texto plano como Telnet y obliga a los administradores a usar túneles cifrados.

Aplica esta ACL Extendida en el CORE L3 (Cisco 3650) para cumplir la rúbrica al 100%. (Reemplaza las IPs con las de tus servidores).

Bloqueo Implícito (Deny Any Any)

Toda ACL tiene un bloqueo automático al final. Si omites el comando permit ip any any como última regla de tu lista, bloquearás absolutamente todo el tráfico normal (pings, etc).

! Entramos a configurar la ACL Extendida CORE_LIMA(config)# ip access-list extended POLITICAS_LIMA ! POLÍTICA 1 - FTP: Permitir TCP (21) solo hacia el Servidor FTP específico (usando host) CORE_LIMA(config-ext-nacl)# permit tcp any host 10.81.43.38 eq 21 ! POLÍTICA 2 - WEB: Permitir tráfico web HTTP (80) y HTTPS (443) CORE_LIMA(config-ext-nacl)# permit tcp any any eq 80 CORE_LIMA(config-ext-nacl)# permit tcp any any eq 443 ! POLÍTICA 3 - DHCP: Permitir servicio de IPs dinámicas en los puertos UDP 67 y 68 CORE_LIMA(config-ext-nacl)# permit udp any any eq 67 CORE_LIMA(config-ext-nacl)# permit udp any any eq 68 ! POLÍTICA 4 - CORREO: Permitir tráfico SMTP (25) y POP3 (110) hacia el Server Mail CORE_LIMA(config-ext-nacl)# permit tcp any host 10.81.43.37 eq 25 CORE_LIMA(config-ext-nacl)# permit tcp any host 10.81.43.37 eq 110 ! CRÍTICO: Evitar que el Deny Implícito bloquee todo lo demás en la red CORE_LIMA(config-ext-nacl)# permit ip any any CORE_LIMA(config-ext-nacl)# exit ! ========================================== ! POLÍTICA 5 - SSH GESTIÓN SEGURA: Crear la clave RSA y el usuario CORE_LIMA(config)# ip domain-name lima.com ! Aquí definimos el "modulus" o tamaño del candado matemático (Digita 1024) CORE_LIMA(config)# crypto key generate rsa CORE_LIMA(config)# username adminLima privilege 15 secret adminLima CORE_LIMA(config)# ip ssh version 2 ! Crear ACL Estándar para asegurar que solo la PC ADMIN acceda CORE_LIMA(config)# ip access-list standard ACL_GESTION CORE_LIMA(config-std-nacl)# permit host 10.81.42.193 (IP de tu PC Admin) CORE_LIMA(config-std-nacl)# exit ! Aplicar la restricción a las líneas virtuales, forzando túnel SSH CORE_LIMA(config)# line vty 0 4 CORE_LIMA(config-line)# transport input ssh CORE_LIMA(config-line)# login local CORE_LIMA(config-line)# access-class ACL_GESTION in

WLAN (Wireless) y Solución Cloud

Para los Access Points (modelo Cisco PT-AC en Packet Tracer), entra a la pestaña "Config" -> Port 1. Asigna el SSID respectivo (ej. WiFi-ClientesVLAN60) y elige autenticación WPA2-PSK con una frase de paso sencilla como 0123456789. Los Smartphones o Laptops EndPoints deben configurarse con el mismo SSID y contraseña en su interfaz Wireless0 para conectarse exitosamente.
