Troubleshooting y Cheatsheet CLI

Aplica el proceso de solución de problemas en capas: comprueba primero los enlaces físicos, luego el direccionamiento, y finalmente el enrutamiento.

Comando Completo Desglose de Parámetros (Flags) y Función Principal

tracert [ip-destino]
(En PC Windows)

(En Router/Switch)		 Rastreo salto por salto. Muestra cada router por el que pasa el paquete. Es vital para encontrar el punto exacto donde se rompe la red.

El parámetro do fuerza al sistema a ejecutar comandos de visualización (show) o guardado sin tener que salirte del modo de configuración (config)# o (config-if)#.

Comando Aplicación y Efecto

Diseño Lógico IP y VLSM (Base del Proyecto)

Antes de hacer tu VLSM: ¿Cuántos hosts necesita Gestión?

Muchos grupos llegan al Módulo 7 (Seguridad SSH) y se dan cuenta de que no les alcanzan las IPs para administrar sus equipos, teniendo que rehacer todo su Excel. La VLAN de Gestión (VLAN 99 para este ejemplo) NO es un número al azar.

Debes asignar 1 IP a CADA equipo de red de tu sucursal para poder controlarlo a distancia. Por lo tanto, el cálculo de hosts actuales de tu VLAN 99 será igual a la suma de:

(Cantidad de Routers(En este caso 1 porque solo hay 1 router por sucursal)) + (Switches L3 de Distribución y el Core) + (Switches L2 de Acceso)

Ejemplo Práctico:

Si tu sucursal tiene 1 Router, 1 Switch Core L3, 3 Switches de Distribución L3 y 5 Switches de Acceso L2, entonces tienes 10 equipos de red.

Tus Hosts Actuales = 10. Le sumas el 25% de crecimiento según la rúbrica (10 * 1.25) = 12.5 (Redondeado a 13 Hosts Proyectados). Necesitarás buscar en el VLSM una subred /28 (14 útiles) o un /27 (30 útiles).

Para tener más claras las cosas a lo largo de esta guía, utilizaremos un subnetting de ejemplo basado exactamente en nuestros archivos de cálculo. Todos los comandos de esta página utilizarán las IPs de esta tabla para que sea más fácil extrapolarlos al trabajo de Packet Tracer.

Sede Principal: LIMA (10.192.8.0/22) VLAN / Uso Dirección de Red IP de Gateway (Primera IP) Máscara de Subred
Ventas VLAN 10 10.192.8.0 /24 10.192.8.1 255.255.255.0
Administración VLAN 20 10.192.9.0 /25 10.192.9.1 255.255.255.128
Finanzas VLAN 30 10.192.9.128 /26 10.192.9.129 255.255.255.192
Nativa-Gestion (SSH) VLAN 99 10.192.11.192 /27 10.192.11.193 255.255.255.224
Nativa (Tráfico sin etiquetar) VLAN 999 No requiere IP (Solo capa 2) N/A N/A

* Nota: Sigue la misma lógica extrayendo los datos de tus archivos CSV para las VLANs de Ica (10.192.12.0/24), La Libertad (10.192.13.0/24), Puno (10.192.14.0/24) y Huánuco (10.192.15.0/24).

No desperdicies IPs en Enlaces de Tránsito o WAN

Tanto para las conexiones WAN (Router a Router) como para las interconexiones internas entre tus switches L3, debes usar SIEMPRE subredes /30 (Máscara 255.255.255.252). Una máscara /30 provee solo 2 direcciones útiles. Para estos ejemplos, usaremos el bloque 172.16.0.0/30 como nuestras IPs de tránsito.

02

Capa 2 - VLANs y Troncales (Cisco 2950-24)

La regla de oro contra ataques (VLAN Hopping)

Por defecto, Cisco usa la VLAN 1 tanto para administrar el switch como para el tráfico "Nativo" (sin etiquetar). Esto es extremadamente inseguro. Por ello, aplicamos la regla de separación:

• VLAN 99 (Nativa-Gestion): Aquí conectaremos la PC del administrador de red y le pondremos una IP al switch para poder acceder a él de forma remota y segura mediante SSH.

• VLAN 999 (Nativa real): Se usará exclusivamente en los cables troncales para atrapar cualquier tráfico basura o malicioso que viaje sin etiquetar y enviarlo a un "agujero negro", ya que esta VLAN no tendrá dirección IP. Recuerda que este es un ejemplo, puedes utilizar cualquier número para la VLAN no solo 999

! 1. CREAR LAS VLANs SECUENCIALMENTE: Switch> enable Switch# configure terminal Switch(config)# vlan 10 Switch(config-vlan)# name Administracion ! Crear la VLAN de Gestión (Para el SSH) Switch(config-vlan)# vlan 99 Switch(config-vlan)# name Nativa-Gestion ! Crear la VLAN Nativa (Agujero negro sin IP) Switch(config-vlan)# vlan 999 Switch(config-vlan)# name Nativa Switch(config-vlan)# exit
! 2. ASIGNAR PUERTO DE ACCESO (Hacia una PC EndPoint) Switch(config)# interface fastEthernet 0/2 Switch(config-if)# switchport mode access Switch(config-if)# switchport access vlan 10 Switch(config-if)# exit
! 3. CONFIGURAR ENLACES TRONCALES (Hacia el Switch L3 correspondiente) Switch(config)# interface gigabitEthernet 0/1 Switch(config-if)# switchport mode trunk ! Declaramos que la VLAN 999 atrapará el tráfico sin etiquetar del troncal: Switch(config-if)# switchport trunk native vlan 999 Switch(config-if)# switchport trunk allowed vlan all Switch(config-if)# exit
! 4. CONFIGURAR GESTIÓN PARA SSH (VLAN 99) ! Le damos una IP al switch en la red de gestión para poder administrarlo a distancia. (Usamos la IP .2 para este switch) Switch(config)# interface vlan 99 Switch(config-if)# ip address 10.192.11.194 255.255.255.224 Switch(config-if)# no shutdown Switch(config-if)# exit ! CRÍTICO: Indicamos la IP de la SVI del Switch L3 de Distribución (.193) para que este Switch L2 sepa cómo responder a la PC Admin que está en otra red. Switch(config)# ip default-gateway 10.192.11.193 Switch(config)# do wr
03

El Núcleo L3 - Modelo Jerárquico (3 Capas)

Nuestra sede matriz cumple estrictamente con el modelo de 3 capas jerárquicas de Cisco. Los switches Multicapa cooperan con funciones separadas y claras:

1. Capa de Distribución (D1_LIMA, D2_LIMA, D3_LIMA)

Aquí es donde configuramos las VLANs de usuario y viven las SVIs (Puertas de enlace de las PCs y APs). Hacia abajo, se conectan a los switches de acceso mediante enlaces troncales L2. Hacia arriba, se conectan al CORE usando enlaces ruteados puros de Capa 3.

2. Capa Core L3 (CORE_LIMA)

Es el agregador central de alta velocidad. NO lleva VLANs de usuario ni SVIs de departamentos. Su única función es unir las conexiones de D1, D2 y D3 con el Router WAN mediante subredes Punto a Punto (/30) y enrutar los paquetes usando RIPv2.

¿Qué hace el flag "no switchport"?

Apaga las funciones de switch (Capa 2) en un puerto físico. Lo convierte en un puerto de Router al que le podemos poner una IP directa, permitiendo enlaces Punto a Punto rápidos hacia el Core sin que el protocolo Spanning Tree nos bloquee la red.

El Secreto del Comando "network" en RIPv2

El comando network NO te pide la IP exacta de tu VLAN (ej. 10.192.8.0), sino la Red Principal (Clase A, B o C) a la que pertenece.

• Como nuestro proyecto usa IPs que empiezan con 10.x.x.x para las PCs, la red principal obligatoria a declarar es 10.0.0.0.

• Para los cables directos entre los switches usaremos IPs 172.16.x.x, por lo que su red principal es 172.16.0.0.

! CONFIGURACIÓN EN D1_LIMA (Asumiendo que maneja VLAN 10 Ventas, VLAN 20 Admin y VLAN 99 Gestión) D1_LIMA(config)# ip routing
! 1. Crear las SVIs (Extraídas de nuestra tabla de Subnetting) D1_LIMA(config)# vlan 10 D1_LIMA(config-vlan)# vlan 20 D1_LIMA(config-vlan)# vlan 99 D1_LIMA(config-vlan)# exit
D1_LIMA(config)# interface vlan 10 D1_LIMA(config-if)# ip address 10.192.8.1 255.255.255.0 (Gateway Ventas) D1_LIMA(config-if)# no shutdown D1_LIMA(config-if)# exit
D1_LIMA(config)# interface vlan 20 D1_LIMA(config-if)# ip address 10.192.9.1 255.255.255.128 (Gateway Admin) D1_LIMA(config-if)# no shutdown D1_LIMA(config-if)# exit
D1_LIMA(config)# interface vlan 99 D1_LIMA(config-if)# ip address 10.192.11.193 255.255.255.224 (Gateway de Gestión) D1_LIMA(config-if)# no shutdown D1_LIMA(config-if)# exit
! 2. Enlace de subida (Uplink) hacia el CORE_LIMA - Puerto Ruteado L3 D1_LIMA(config)# interface gigabitEthernet 1/0/24 D1_LIMA(config-if)# no switchport ! Usamos una IP para el cable que los une (Subred /30 de Tránsito) D1_LIMA(config-if)# ip address 172.16.0.2 255.255.255.252 D1_LIMA(config-if)# no shutdown D1_LIMA(config-if)# exit
! 3. Configurar RIPv2 para anunciar las redes locales hacia el Core D1_LIMA(config)# router rip D1_LIMA(config-router)# version 2 ! Obligatorio: Activa RIP en todas tus VLANs de usuarios (Las que empiezan con 10) D1_LIMA(config-router)# network 10.0.0.0 ! Obligatorio: Activa RIP en el cable /30 que acabas de configurar (Empieza con 172.16) D1_LIMA(config-router)# network 172.16.0.0 D1_LIMA(config-router)# no auto-summary
! Nota: Configura D2_LIMA y D3_LIMA siguiendo exactamente esta misma lógica para sus respectivas VLANs.

El CORE_LIMA interconecta los tres bloques de distribución ruteando los paquetes y enviándolos hacia el Router 2911 Perimetral de salida. No tiene SVIs, solo puertos ruteados IP.

! CONFIGURACIÓN EN CORE_LIMA CORE_LIMA(config)# ip routing
! 1. Conexión hacia el Distribución 1 (Mismo segmento Punto a Punto) CORE_LIMA(config)# interface gigabitEthernet 1/0/1 CORE_LIMA(config-if)# no switchport CORE_LIMA(config-if)# ip address 172.16.0.1 255.255.255.252 CORE_LIMA(config-if)# no shutdown CORE_LIMA(config-if)# exit
! (Repetir bloque anterior para los puertos que van a D2 y D3 usando diferentes IPs /30 como 172.16.0.5 y 172.16.0.9)
! 2. Conexión hacia el Router 2911 Perimetral CORE_LIMA(config)# interface gigabitEthernet 1/0/24 CORE_LIMA(config-if)# no switchport CORE_LIMA(config-if)# ip address 172.16.0.13 255.255.255.252 CORE_LIMA(config-if)# no shutdown CORE_LIMA(config-if)# exit
! 3. Activar RIPv2 en el Core para unir los caminos CORE_LIMA(config)# router rip CORE_LIMA(config-router)# version 2 ! El Core NO tiene VLANs de PCs conectadas, así que solo anuncia la red 172.16.0.0 que engloba a todos sus cables /30 CORE_LIMA(config-router)# network 172.16.0.0 CORE_LIMA(config-router)# no auto-summary
04

Enlaces WAN (Routers 2911) y Autenticación PPP

Fundamento Teórico

En enlaces seriales WAN, para aplicar seguridad cambiamos el encapsulamiento a PPP (Point-to-Point Protocol).

💡 Analogía simple: Piensa en PAP como enseñar tu DNI en la puerta del edificio (cualquiera puede ver tus datos, va en texto plano). En cambio, CHAP es como usar un token de banco; envía un desafío matemático cifrado que cambia constantemente.
Distribución Obligatoria (WAN 1 al 4)

La rúbrica exige usar AMBOS protocolos. Configuraremos CHAP para conectar Lima con La Libertad y Huánuco (WAN 1 y 3). Usaremos PAP para conectar Lima con Ica y Puno (WAN 2 y 4).

! ========================================== ! ENLACES CON PAP (Ej: Lima conectando hacia Ica - WAN 2) ! ========================================== ! username [host_vecino] secret [clave_compartida] Router_Lima(config)# username Router_Ica secret ciscoclass Router_Lima(config)# interface s0/0/0 ! Usa una subred WAN reservada como 10.192.255.1 /30 (A modo de ejemplo) Router_Lima(config-if)# ip address 10.192.255.1 255.255.255.252 Router_Lima(config-if)# encapsulation ppp Router_Lima(config-if)# ppp authentication pap ! PAP requiere enviar explícitamente las credenciales propias: Router_Lima(config-if)# ppp pap sent-username Router_Lima password ciscoclass Router_Lima(config-if)# no shutdown
! ========================================== ! ENLACES CON CHAP (Ej: Lima conectando hacia La Libertad - WAN 1) ! ========================================== ! CHAP es automático, confía en la BD local de usuarios. Router_Lima(config)# username Router_LaLibertad secret ciscoclass Router_Lima(config)# interface s0/0/1 Router_Lima(config-if)# encapsulation ppp Router_Lima(config-if)# ppp authentication chap Router_Lima(config-if)# no shutdown

Enrutamiento RIPv2, Failover e Inyección de Ruta

Comando y Flags ¿Qué hace exactamente?
ip route 0.0.0.0 0.0.0.0 [ip-isp] [distancia] Ruta Estática Predeterminada. Le dice al Router Perimetral: "Si no conoces el destino del paquete, tíralo hacia Internet". El parámetro opcional [distancia] crea una ruta flotante de respaldo que solo se activará si la principal falla.
default-information originate COMANDO CRÍTICO. Como las sucursales no tienen ruta hacia Internet, este comando dentro de router rip obliga al Router Perimetral a inyectar su ruta estática predeterminada dentro de las actualizaciones RIPv2, para que todos los demás routers de la topología aprendan cómo salir a Internet.
no auto-summary Desactiva la sumarización automática que agrupa redes a su clase mayor. Si no lo pones, RIPv2 destruirá tu diseño VLSM mezclando las subredes.
! 1. RUTAS ESTÁTICAS HACIA EL INTERNET (Solo en el Router Perimetral 2911): ! Ruta Predeterminada Principal por el ISP 1 (Distancia predeterminada = 1) Router_Lima(config)# ip route 0.0.0.0 0.0.0.0 209.165.200.226 ! Ruta Flotante de Respaldo por el ISP 2 (Agregamos la Distancia Administrativa 5) Router_Lima(config)# ip route 0.0.0.0 0.0.0.0 209.165.201.2 5
! 2. ENRUTAMIENTO DINÁMICO E INYECCIÓN Router_Lima(config)# router rip Router_Lima(config-router)# version 2 ! Declaramos las redes de clase conectadas a este router (WANs hacia sucursales y la interfaz ruteada hacia el CORE) Router_Lima(config-router)# network 10.0.0.0 Router_Lima(config-router)# network 172.16.0.0 Router_Lima(config-router)# no auto-summary ! Inyectamos la ruta de internet al resto de la topología: Router_Lima(config-router)# default-information originate

Servicios de Red (La Magia de Capa 7)

Para configurar correctamente las políticas de seguridad (ACLs), necesitas saber bajo qué protocolo de transporte y puerto se comunica cada servicio:

Servicio (App) Capa de Transporte y Puerto Descripción de uso
HTTP / HTTPS (WEB) TCP 80 / 443 Conexiones orientadas a garantizar la entrega de páginas web al navegador.
FTP (Archivos) TCP 20 y 21 Garantiza la fiabilidad del envío de paquetes. Usa el puerto 21 para control y 20 para datos.
SMTP (Correo Saliente) TCP 25 Se encarga del envío del mensaje (MTA) desde el cliente al servidor.
POP3 (Correo Entrante) TCP 110 Se encarga de descargar y recuperar el correo desde el servidor hacia la PC cliente.
DNS (Nombres) UDP 53 Traduce dominios a IPs. Usa UDP porque requiere una respuesta rápida.
DHCP (IP Dinámica) UDP 67 y 68 Asignación rápida de red para clientes (puerto 68) y respuestas del servidor (puerto 67).
¡El Gateway del DHCP es Crítico!

Cuando configures los Pool de DHCP para tus VLANs desde el Servidor GUI, en el campo "Default Gateway" debes colocar OBLIGATORIAMENTE la IP de la SVI del Switch de Distribución L3 encargado de esa subred, de lo contrario las PCs finales no podrán salir de su red.

ServicioPaso a Paso en Packet Tracer GUI
DNS Enciende. En Name escribe el dominio (ej. `miempresa.com`), en Type elige `A Record`, y en Address pon la IP exacta del servidor respectivo.
HTTP Enciende HTTP y HTTPS. Edita el archivo `index.html` haciendo clic en "Edit".
FTP Actívalo (ON). Crea el usuario (ej. `adminLima`), contraseña y marca los permisos: Read, Write, Delete, Rename, List. Clic en "Add".
EMAIL Enciende SMTP y POP3. En Domain Name escribe `mail.miempresa.com` y dale a "Set". Crea los usuarios para que las PCs puedan enviarse correos usando la App nativa.
07

Seguridad, ACLs Extendidas y SSH

La receta para habilitar el Acceso Remoto Seguro (SSH)

Para que el protocolo SSH funcione, el equipo requiere 4 cosas obligatorias antes de generar las llaves criptográficas: 1) Un Hostname distinto al de fábrica. 2) Un nombre de dominio. 3) Un usuario local con contraseña secreta. 4) Configurar las líneas virtuales (VTY) para que acepten el protocolo.

! ========================================== ! EJEMPLO A: CONFIGURAR SSH EN EL ROUTER 2911 ! ========================================== Router(config)# hostname Router_Lima Router_Lima(config)# ip domain-name miempresa.com ! Generar llave. Te preguntará el tamaño: Escribe 1024 o 2048 Router_Lima(config)# crypto key generate rsa Router_Lima(config)# username adminLima privilege 15 secret ciscoclass Router_Lima(config)# ip ssh version 2 ! Entramos a las líneas virtuales (del 0 al 4 son 5 sesiones simultáneas) Router_Lima(config)# line vty 0 4 Router_Lima(config-line)# transport input ssh (Mata las conexiones Telnet) Router_Lima(config-line)# login local (Obliga a usar el usuario que creamos) Router_Lima(config-line)# exit
! ========================================== ! EJEMPLO B: CONFIGURAR SSH EN UN SWITCH MULTICAPA L3 (Core o Distribución) ! ========================================== ! Los switches Multicapa necesitan tener su SVI de Gestión activa para ser alcanzables. Switch(config)# hostname D1_LIMA D1_LIMA(config)# interface vlan 99 D1_LIMA(config-if)# ip address 10.192.11.193 255.255.255.224 D1_LIMA(config-if)# no shutdown D1_LIMA(config-if)# exit D1_LIMA(config)# ip domain-name miempresa.com D1_LIMA(config)# crypto key generate rsa D1_LIMA(config)# username adminLima privilege 15 secret ciscoclass D1_LIMA(config)# ip ssh version 2 ! Los switches soportan más conexiones simultáneas (0 a 15) D1_LIMA(config)# line vty 0 15 D1_LIMA(config-line)# transport input ssh D1_LIMA(config-line)# login local D1_LIMA(config-line)# exit
! ========================================== ! EJEMPLO C: CONFIGURAR SSH EN UN SWITCH L2 DE ACCESO ! ========================================== Switch(config)# hostname SW_ACCESO1 SW_ACCESO1(config)# interface vlan 99 SW_ACCESO1(config-if)# ip address 10.192.11.194 255.255.255.224 SW_ACCESO1(config-if)# no shutdown SW_ACCESO1(config-if)# exit ! CRÍTICO EN L2: Sin el Default-Gateway, la PC Admin que está en la VLAN 20 no podrá hacerle SSH a la VLAN 99 del Switch. SW_ACCESO1(config)# ip default-gateway 10.192.11.193 SW_ACCESO1(config)# ip domain-name miempresa.com SW_ACCESO1(config)# crypto key generate rsa SW_ACCESO1(config)# username adminLima privilege 15 secret ciscoclass SW_ACCESO1(config)# line vty 0 15 SW_ACCESO1(config-line)# transport input ssh SW_ACCESO1(config-line)# login local SW_ACCESO1(config-line)# exit

Las Listas de Acceso (ACL) necesitan aplicarse

Muchos proyectos fallan la evaluación porque configuran una ACL perfectamente bien, pero se olvidan de enlazarla a una interfaz. Una ACL es solo una lista de reglas abstractas hasta que usas el comando ip access-group [nombre-de-la-acl] [in / out] dentro de una interfaz de red.

• El flag in (Entrada): Filtra el tráfico justo en el momento en el que el paquete "intenta entrar" al equipo por ese puerto.

• El flag out (Salida): Deja que el equipo procese y enrute el paquete, pero lo evalúa y frena justo en el momento en que "intenta salir" por ese puerto hacia su destino.

Crearemos la ACL Extendida en el CORE L3. Al final, usaremos el flag in sobre el puerto físico que conecta con el exterior para filtrar peticiones entrantes no autorizadas.

! 1. CREACIÓN DE LA LISTA EXTENDIDA CORE_LIMA(config)# ip access-list extended POLITICAS_LIMA ! POLÍTICA 1 y 2 - Permitir FTP (21) y WEB (80) hacia los servidores locales (usando IPs de la Sede) CORE_LIMA(config-ext-nacl)# permit tcp any host 10.192.11.10 eq 21 (Ejemplo IP Servidor FTP) CORE_LIMA(config-ext-nacl)# permit tcp any host 10.192.11.11 eq 80 (Ejemplo IP Servidor Web) ! POLÍTICA 3 - DHCP (Puertos UDP 67 y 68) CORE_LIMA(config-ext-nacl)# permit udp any any eq 67 CORE_LIMA(config-ext-nacl)# permit udp any any eq 68 ! POLÍTICA 4 - CORREO (Tráfico SMTP 25 y POP3 110) CORE_LIMA(config-ext-nacl)# permit tcp any host 10.192.11.12 eq 25 (Ejemplo IP Mail Server) CORE_LIMA(config-ext-nacl)# permit tcp any host 10.192.11.12 eq 110 ! Evitar que el Deny Implícito bloquee todo lo demás (Pings) CORE_LIMA(config-ext-nacl)# permit ip any any CORE_LIMA(config-ext-nacl)# exit
! 2. APLICACIÓN DE LA POLÍTICA (El paso crítico) ! Entramos al puerto físico ruteado que conecta directamente con el Router WAN 2911 CORE_LIMA(config)# interface gigabitEthernet 1/0/24 CORE_LIMA(config-if)# ip access-group POLITICAS_LIMA in CORE_LIMA(config-if)# exit

WLAN (Wireless) y Solución Cloud

Para los Access Points (modelo Cisco PT-AC), entra a la pestaña "Config" -> Port 1. Asigna el SSID respectivo (ej. WiFi-Clientes) y elige autenticación WPA2-PSK con una frase de paso sencilla como 0123456789. Los Smartphones deben configurarse con el mismo SSID y contraseña en su interfaz Wireless0.
